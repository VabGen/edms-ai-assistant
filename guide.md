## 1. Что такое EDMS AI Assistant

**EDMS AI Assistant** — это интеллектуальный помощник, встроенный в систему электронного документооборота. Он понимает запросы на естественном русском языке и помогает выполнять повседневные задачи с документами без ручного поиска по меню.

Вы общаетесь с ним как в обычном чате: пишете вопрос или задание — ассистент выполняет его и отвечает по-русски.

**Ключевые принципы работы:**
- Понимает свободный текст — не нужно знать команды или синтаксис
- Всегда работает в контексте открытого документа — не нужно указывать номер документа вручную
- Отвечает структурированно: списки, выделение дат, сумм, имён
- Никогда не показывает технические данные (коды, идентификаторы) — только понятную информацию

## 2. Как начать работу

1. Откройте документ в системе EDMS
2. Найдите панель чата с ассистентом (обычно справа или внизу экрана)
3. Напишите свой вопрос или задание в поле ввода
4. Нажмите Enter или кнопку отправки

Ассистент автоматически видит, какой документ у вас открыт, и использует его как контекст для ответа.

> **Совет:** Можно также прикрепить файл (например, Word-документ или PDF) — ассистент прочитает его содержимое и ответит на вопросы по нему.

## 3. Полный список возможностей

### 3.1 Получение информации о документе

Ассистент читает все поля карточки открытого документа и отвечает на вопросы о нём.

**Что можно узнать:**

| Информация                   | Пример запроса                              |
|------------------------------|---------------------------------------------|
| Основные реквизиты           | «Покажи основную информацию по документу»   |
| Статус и этап                | «Какой статус у документа?»                 |
| Автор и подписант            | «Кто автор документа?» / «Кем подписан?»    |
| Корреспондент / контрагент   | «Кто контрагент по договору?»               |
| Регистрационный номер и дата | «Какой номер у этого документа?»            |
| Краткое содержание           | «О чём этот документ?»                      |
| Поручения и исполнители      | «Какие поручения есть по документу?»        |
| Этапы процесса (маршрут)     | «На каком этапе находится документ?»        |
| Контроль                     | «Стоит ли документ на контроле?»            |
| Вложения                     | «Какие файлы прикреплены к документу?»      |
| Ответственные (договор)      | «Кто ответственный по договору?»            |
| Данные договора              | «Какая сумма договора?» / «Дата окончания?» |
| Данные обращения             | «Кто заявитель?» / «Тип обращения?»         |
| Лист ознакомления            | «Кто ознакомился с документом?»             |

---

### 3.2 Поиск документов

Ищет документы в базе EDMS по различным критериям.

**Параметры поиска:**
- Текстовый запрос (по содержанию, автору, теме)
- Регистрационный номер (точный или частичный)
- Категория документа: Внутренний, Входящий, Исходящий, Обращение, Договор, Совещание
- Диапазон дат регистрации

**Примеры:**
```
Найди документы про договор с поликлиникой
Есть ли входящие документы за март 2026?
Найди документ с номером ВХ-2026-015
Покажи все договоры за последний квартал
```

---

### 3.3 Чтение и анализ вложений

Извлекает текст из прикреплённых файлов и отвечает на вопросы по их содержимому.

**Поддерживаемые форматы:** PDF, Word (DOCX), текстовые файлы.

**Примеры:**
```
Прочитай вложенный договор и скажи, какие штрафные санкции предусмотрены
Что написано в акте приёма-передачи?
Есть ли в документе упоминание суммы 50 000 рублей?
```

---

### 3.4 Создание краткого изложения (суммаризация)

Сжимает длинный текст документа или вложения в удобный формат. Доступно три режима:

| Режим               | Описание                                               | Когда использовать               |
|---------------------|--------------------------------------------------------|----------------------------------|
| **Выжимка фактов**  | Только ключевые факты: суммы, даты, участники, решения | Нужна быстрая справка            |
| **Тезисный разбор** | Структурированные тезисы по смыслу документа           | Нужно понять суть и структуру    |
| **Пересказ**        | Связный краткий пересказ своими словами                | Нужно ознакомиться с содержанием |

**Примеры:**
```
Сделай краткое изложение документа
Тезисно перескажи содержимое договора
Выдели ключевые факты из акта
```

---

### 3.5 Сравнение версий документа

Анализирует все версии документа и показывает что изменилось между ними.

**Примеры:**
```
Что изменилось в документе по сравнению с предыдущей версией?
Сравни версии документа
Покажи изменения между версиями
```

---

### 3.6 Сравнение файла с вложением

Если вы загрузили файл в чат — ассистент сравнит его с вложением документа и покажет различия.

**Примеры:**
```
Сравни мой файл с договором из системы
Есть ли отличия между загруженным файлом и вложением?
```

---

### 3.7 Создание поручения

Создаёт поручение по открытому документу с указанием текста, исполнителей и срока.

**Что нужно указать:**
- Текст поручения (обязательно)
- Фамилия исполнителя или нескольких исполнителей (обязательно)
- Срок исполнения (необязательно; если не указан — автоматически +7 дней)
- Ответственный исполнитель (необязательно; если не указан — первый из списка)

**Примеры:**
```
Создай поручение Иванову: подготовить ответ на письмо до 20 апреля
Поставь задачу Петрову и Сидорову — проверить договор, срок неделя
Создай поручение: согласовать смету. Исполнитель — Козлов, срочно
```

> **Уточнение:** Если по фамилии найдено несколько сотрудников — ассистент покажет список и попросит уточнить нужного.

---

### 3.8 Добавление в лист ознакомления

Добавляет указанных сотрудников в лист ознакомления с документом.

**Примеры:**
```
Добавь Иванова в лист ознакомления
Ознакомь с документом Петрова и Сидорову
Добавь меня в лист ознакомления
```

---

### 3.9 Отправка уведомлений и напоминаний

Отправляет уведомления сотрудникам через систему EDMS.

**Типы уведомлений:**
- **Напоминание** — общее напоминание по документу
- **Срок** — уведомление о приближающемся дедлайне
- **Произвольное** — любой текст

**Примеры:**
```
Напомни Иванову о необходимости подписать договор до пятницы
Отправь уведомление Петрову: документ ожидает согласования
```

---

### 3.10 Поиск сотрудников

Находит информацию о сотрудниках организации по фамилии.

**Примеры:**
```
Найди сотрудника Иванова
Кто такой Петров Александр?
Найди главного бухгалтера Козлову
```

---

### 3.11 Автозаполнение карточки обращения

Для документов категории **«Обращение граждан»** — автоматически заполняет поля карточки на основе анализа прикреплённого файла (заявления, письма).

**Что заполняется автоматически:**
- ФИО и тип заявителя
- Адрес (регион, район, город)
- Краткое содержание
- Вид обращения и способ доставки

**Пример:**
```
Заполни карточку обращения по прикреплённому заявлению
Автозаполни документ
```

## 4. Примеры диалогов и фразы для запросов

## 5. Советы — как лучше формулировать запросы

### ✅ Пишите конкретно

| Плохо                 | Хорошо                                                     |
|-----------------------|------------------------------------------------------------|
| «Поручение Иванову»   | «Создай поручение Иванову: подготовить отчёт до 20 апреля» |
| «Найди документ»      | «Найди входящие письма от ООО Альфа за март»               |
| «Сделай краткое»      | «Сделай выжимку фактов из договора»                        |
| «Добавь в список»     | «Добавь Петрова и Сидорову в лист ознакомления»            |

---

### ✅ Указывайте срок при создании поручения

Ассистент понимает любой формат даты:

```
до 15 апреля
к концу месяца
через две недели
срочно (будет установлено +7 дней автоматически)
```

---

### ✅ Используйте фамилии сотрудников

Ассистент ищет сотрудников по фамилии. Если нашлось несколько — уточнит. Можно также написать «добавь меня» — ассистент знает вашу фамилию автоматически.

---

### ✅ Задавайте уточняющие вопросы

После любого ответа можно задать уточняющий вопрос:

```
А какой срок у первого поручения?
Расскажи подробнее о контрагенте
Что именно изменилось во второй версии?
```

---

### ✅ Загружайте файлы для анализа

Если нужно сравнить внешний документ с тем, что в системе, или задать вопрос по файлу с компьютера — просто прикрепите файл в чат и спросите:

```
[прикрепляет файл] Сравни этот файл с договором из системы
[прикрепляет файл] Что написано в этом акте?
```

---

### ✅ Не нужно указывать номер документа

Ассистент автоматически работает с тем документом, который открыт у вас в системе. Указывать ID или номер вручную не требуется.

---

### ✅ Если ответ неточный — перефразируйте

Попробуйте уточнить или добавить деталей:

```
Вместо: «Покажи поручения»
Написать: «Покажи список поручений по этому документу — кто исполнитель и какой срок»
```

## 6. Ограничения — что ассистент не умеет

### Не выполняет действия вне своих инструментов

Ассистент не может:
- Редактировать текст самого документа в системе
- Менять статус документа вручную (зарегистрировать, завершить маршрут)
- Удалять документы или поручения
- Отправлять электронные письма во внешние почтовые системы
- Распечатывать документы
- Выгружать документы в Excel или другие форматы

---

### Не работает с документами вне системы EDMS

Ассистент видит только те документы, которые находятся в вашей системе электронного документооборота. Он не может:
- Искать информацию в интернете
- Работать с документами из других систем (1С, SharePoint и т.д.)
- Получать данные из внешних баз

---

### Не выполняет несколько несвязанных задач в одном сообщении

Лучше разбивать сложные задачи на шаги:

```
❌ «Найди договор с поликлиникой, создай по нему поручение Иванову и добавь Петрова в ознакомление»

✅ Шаг 1: «Найди договор с 10-й городской поликлиникой»
✅ Шаг 2: «Создай по нему поручение Иванову: проверить исполнение, срок 30 апреля»
✅ Шаг 3: «Добавь Петрова в лист ознакомления»
```

---

### Автозаполнение обращений — только для категории «Обращение»

Функция автозаполнения карточки работает только с документами категории **«Обращение граждан»**. Для других категорий она недоступна.

---

### Ответы основаны на данных в системе

Если информация в карточке документа не заполнена или устарела — ассистент отобразит то, что есть в системе. Он не дополняет данные из других источников.

---

### Ответы только на русском языке

Ассистент отвечает исключительно на русском языке. Вопросы можно задавать в любой формулировке — ответ всегда будет на русском.

---

## 7. Частые вопросы (FAQ)

**В: Нужно ли каждый раз указывать, с каким документом работать?**  
О: Нет. Ассистент автоматически видит открытый у вас документ.

---

**В: Что делать, если ассистент нашёл не того сотрудника?**  
О: Если по фамилии найдено несколько человек, ассистент покажет список и попросит уточнить. Просто ответьте «первого», «второго» или укажите полное имя.

---

**В: Можно ли отменить действие (например, созданное поручение)?**  
О: Ассистент не отменяет выполненные операции. Для отмены поручения используйте стандартный интерфейс системы EDMS.

---

**В: Ассистент создал поручение — почему оно не отображается сразу?**  
О: После создания поручения или другого изменения страница может обновиться автоматически. Если этого не произошло — обновите страницу вручную (F5).

---

**В: Почему ассистент не читает содержимое некоторых файлов?**  
О: Поддерживаются форматы PDF, Word (DOCX) и текстовые файлы. Сканированные изображения (JPG, PNG), защищённые паролем PDF и файлы в форматах XLS, XLSX не поддерживаются для чтения содержимого.

---

**В: Могу ли я задавать вопросы не по конкретному документу?**  
О: Да. Ассистент ответит на общие вопросы о работе системы, поможет найти нужного сотрудника или подскажет, как выполнить задачу.
